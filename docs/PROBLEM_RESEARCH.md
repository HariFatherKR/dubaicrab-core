## 1. 현재 제품 상태 분석

### 1.1 코드베이스 구조

```
dubaicrab-core/          # 백엔드 (OpenClaw 포크)
├── src/
│   ├── agents/          # AI 에이전트 엔진
│   ├── channels/        # Telegram, Discord, Slack
│   ├── gateway/         # 게이트웨이 서버
│   ├── cli/             # CLI 인터페이스
│   └── skills/          # 스킬 엔진
├── skills/              # 한국 오피스 스킬
│   ├── hwp-parsing/     # HWP 파싱 스킬
│   ├── document-drafting/
│   └── email-writing/
└── dist/                # 빌드 결과

Dubai Crab/                # Tauri UI (데스크톱 앱)
├── src/
│   ├── lib/             # Svelte 컴포넌트
│   └── routes/          # SvelteKit 라우트
├── src-tauri/           # Rust 백엔드
└── package.json

dubaicrab-web/           # 랜딩 페이지 (Next.js)
└── src/
```

## 2. 기술적 갭 분석

### 2.1 Critical Gaps (🔴 P0)

#### Gap 1: HWP 파싱 한계

```
현재 상태:
- pyhwp 기반 기본 텍스트 추출만 가능
- 표, 이미지, 스타일 정보 손실
- HWPX (Open XML) 지원 제한적

필요한 것:
- 완전한 HWP/HWPX 구조 파싱
- 표/이미지 추출 및 이해
- 문서 생성 기능 (역방향)

영향:
- 핵심 차별화 기능 미완성
- 기업 PoC 진행 어려움
```

#### Gap 2: 로컬 LLM 통합 불안정

```
현재 상태:
- Ollama 기본 연동만 구현
- 모델 자동 설치 미지원
- 메모리/성능 최적화 부족

필요한 것:
- 원클릭 모델 설치
- GPU 자동 감지 및 최적화
- 한국어 모델 번들링

영향:
- 설치 복잡도 → 이탈률 증가
- 비개발자 사용 어려움
```

#### Gap 3: Tauri UI 미완성

```
현재 상태:
- 기본 채팅 UI만 구현
- 설정/온보딩 미구현
- 문서 뷰어 없음

필요한 것:
- 완전한 설정 UI
- 가이드 온보딩
- 파일 드래그앤드롭
- 문서 미리보기

영향:
- 사용자 경험 저하
- CLI 의존 → 진입장벽
```

### 2.2 High Priority Gaps (🟠 P1)

#### Gap 4: KakaoTalk 채널 부재

```
현재 상태: 미구현
필요한 것: 카카오 채널 API 연동
영향: 한국 시장 핵심 채널 누락
```

#### Gap 5: 기업 관리 기능 부재

```
현재 상태:
- 단일 사용자 모드만
- 중앙 관리 없음
- 사용량 추적 없음

필요한 것:
- 멀티 테넌트 지원
- 관리자 대시보드
- 사용량/비용 모니터링
```

#### Gap 6: 문서 생성 기능 미구현

```
현재 상태: 텍스트 출력만
필요한 것:
- HWP/DOCX 문서 생성
- 템플릿 기반 공문 작성
- PDF 내보내기
```

## 3. UX/기능 부족 분석

### 3.1 온보딩 UX 문제

```
현재 플로우:
1. npm install -g dubaicrab-core  ← 개발자만 이해
2. dubaicrab config              ← CLI 명령어
3. Ollama 별도 설치              ← 추가 설치 필요
4. 모델 다운로드                  ← 시간 소요
5. dubaicrab gateway start       ← 서버 시작

문제점:
- 5단계 이상 필요 → 이탈률 높음
- 비개발자 진입 불가
- 에러 발생 시 해결 어려움
```

**개선 방향**:

```
이상적인 플로우:
1. dubaicrab.exe 다운로드
2. 더블클릭 실행
3. 원클릭 설정 완료
```

### 3.2 일상 사용 UX 문제

| 영역          | 문제                | 영향도 |
| ------------- | ------------------- | ------ |
| 파일 업로드   | 드래그앤드롭 미지원 | 높음   |
| 문서 미리보기 | 없음                | 높음   |
| 대화 기록     | 검색 불가           | 중간   |
| 단축키        | 미구현              | 낮음   |
| 다크모드      | 미구현              | 낮음   |

### 3.3 기능 부족 영역

## 5. 사용자 여정 분석

### 5.1 현재 사용자 여정 (Pain Points 표시)

```
[인지] "ChatGPT 못 쓰는데 대안 있나?"
    ↓
[검색] 구글 "로컬 AI 한국" ← ❌ Dubai Crab 검색 안됨
    ↓
[발견] GitHub 또는 지인 추천
    ↓
[설치] README 따라 설치 ← ⚠️ 복잡함, 에러 발생
    ↓
[설정] config 파일 편집 ← ⚠️ 개발자 아니면 어려움
    ↓
[시작] gateway 시작 ← ⚠️ CLI 명령어
    ↓
[사용] Telegram 연결 ← ✅ 잘 작동
    ↓
[확장] HWP 파일 업로드 ← ⚠️ 파싱 제한
    ↓
[이탈 or 유지]
```

### 5.2 이상적인 사용자 여정

```
[인지] 광고/추천 "회사에서 AI 쓰고 싶으면 Dubai Crab"
    ↓
[랜딩] dubaicrab.io 방문 → 30초 영상
    ↓
[다운로드] 원클릭 다운로드
    ↓
[설치] 인스톨러 실행 (자동 Ollama 포함)
    ↓
[온보딩] 마법사로 Telegram 연결
    ↓
[첫 사용] "안녕하세요" → AI 응답
    ↓
[가치 확인] HWP 파일 분석 성공
    ↓
[습관화] 매일 사용
    ↓
[확산] 동료에게 추천
```
