## 2. 기능 우선순위 (RICE 분석)

### 2.1 RICE 스코어 계산

**RICE 공식**: `(Reach × Impact × Confidence) / Effort`

- **Reach**: 영향받는 사용자 수 (1-10)
- **Impact**: 사용자당 영향도 (0.5=Low, 1=Medium, 2=High, 3=Massive)
- **Confidence**: 확신도 (0.5-1.0)
- **Effort**: 개발 인력-월 (person-months)

#### F08: RAG 파이프라인

**아키텍처**:

```
┌──────────────────────────────────────────────────────────┐
│                    Document Ingestion                    │
├──────────────────────────────────────────────────────────┤
│  Upload → Chunk → Embed → Store (SQLite + vec extension) │
└──────────────────────────────────────────────────────────┘
                            │
                            ▼
┌──────────────────────────────────────────────────────────┐
│                     Query Pipeline                       │
├──────────────────────────────────────────────────────────┤
│  Question → Embed → Vector Search → Context → LLM → Answer│
└──────────────────────────────────────────────────────────┘
```

**기술 선택**:
| 컴포넌트 | 선택지 | 추천 | 이유 |
|----------|--------|------|------|
| 임베딩 모델 | nomic-embed, BGE-M3 | BGE-M3 | 다국어 우수 |
| 벡터 DB | SQLite-vec, Chroma | SQLite-vec | 로컬 경량 |
| 청킹 | LangChain, Custom | Custom | 한글 최적화 |

**수락 기준**:
