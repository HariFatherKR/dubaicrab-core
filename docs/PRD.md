## 1. 문제 선언문 (Problem Statement)

### 1.1 문제 정의

> **"ChatGPT를 차단하는 한국 기업의 직원들은 AI 생산성 도구를 사용할 수 없어,
> 경쟁사 대비 업무 효율이 저하되고 있다."**

### 1.2 문제 컨텍스트

**배경**:

- 한국 100대 기업 중 60% 이상이 ChatGPT 등 외부 AI 서비스 차단
- 보안 정책, 데이터 주권, 규제 준수 등 다양한 이유
- 직원들은 개인 디바이스로 우회 사용 → 보안 리스크 증가

**영향받는 사용자**:

- 대기업 사무직 (500만명+)
- 공공기관 직원 (100만명+)
- 금융권/법무법인/의료기관 (100만명+)

**현재 대안의 문제**:

- MS Copilot: 클라우드 기반, 데이터 외부 전송
- 로컬 LLM (Ollama, LM Studio): HWP 미지원, 설정 복잡
- 국내 AI (네이버, 카카오): 여전히 클라우드 기반

### 1.3 문제의 심각성

| 측면        | 현재 상태      | 비용              |
| ----------- | -------------- | ----------------- |
| 생산성 손실 | AI 도구 미사용 | 직원당 연 200시간 |
| 보안 리스크 | 우회 사용      | 데이터 유출 위험  |
| 기회 비용   | AI 경쟁력 저하 | 측정 불가         |

## 3. 타겟 사용자 (Target Users)

### 3.1 주요 페르소나

#### 페르소나 1: 김대리 (30, 대기업 사무직)

```
직업: 대기업 기획팀 대리
업무: 보고서 작성, 데이터 분석, 회의 정리
Pain Point:
- "ChatGPT 쓰고 싶은데 회사에서 막혀있어요"
- "HWP 파일 다루는 일이 많은데 AI가 안 읽어요"
- "보고서 초안 잡는데 시간이 너무 많이 걸려요"
Goals:
- 업무 시간 절약
- 보고서 품질 향상
- 팀 내 AI 선도자 되기
```

#### 페르소나 2: 박팀장 (45, 공공기관 팀장)

```
직업: 공공기관 정보화팀장
업무: IT 정책 수립, 보안 관리, 시스템 도입
Pain Point:
- "AI 도구 도입하고 싶은데 보안 이슈가 걱정"
- "직원들이 개인폰으로 ChatGPT 쓰는 게 더 위험"
- "온프레미스 솔루션 찾는데 HWP 되는 게 없어"
Goals:
- 안전한 AI 도입
- 직원 생산성 향상
- 보안 감사 통과
```

#### 페르소나 3: 이개발자 (28, 스타트업 개발자)

```
직업: 스타트업 백엔드 개발자
업무: 코드 작성, 코드 리뷰, 문서화
Pain Point:
- "Copilot 쓰고 싶은데 코드 유출 우려"
- "API 문서 작성이 너무 귀찮아"
- "오픈소스 도구들 설정이 복잡해"
Goals:
- 코딩 속도 향상
- 보안 유지하면서 AI 사용
- 쉬운 설치와 사용
```
