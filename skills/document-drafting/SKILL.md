# 문서 작성

## Overview

한국 기업/공공기관 스타일의 공식 문서를 작성합니다.

## When to Use

- "공문 작성해줘", "보고서 써줘"
- "기안서 양식으로", "품의서 형식으로"
- 상급자 보고용 문서 필요 시

## 문서 유형별 구조

### 공문 (公文)

```
[기관명/부서명]

문서번호: 제2026-001호
시행일자: 2026년 2월 8일
수    신: [수신처]
참    조: [참조처]
제    목: [제목]

1. 귀 기관(부서)의 무궁한 발전을 기원합니다.

2. [본문 내용]

3. 붙임: [첨부파일 목록]

[직급] [이름] (인)
```

### 보고서

```
[제목]

Ⅰ. 개요
  1. 목적
  2. 배경

Ⅱ. 현황 분석
  1. [현황 1]
  2. [현황 2]

Ⅲ. 문제점 및 개선방안
  1. 문제점
  2. 개선방안

Ⅳ. 기대효과

Ⅴ. 향후 계획

※ 붙임: [첨부 목록]
```

### 기안서/품의서

```
기 안 서

기안일자: 2026.02.08
기안부서: [부서명]
기 안 자: [이름] (인)
결    재: [결재라인]

제    목: [제목]

1. 목적

2. 내용

3. 예산
   - 항목: 금액

4. 기대효과

5. 붙임
```

## 작성 원칙

### 문체
- 경어체 사용 (~합니다, ~입니다)
- 간결하고 명확하게
- 주어 생략 가능 (한국어 특성)

### 숫자 표기
- 금액: 1,000,000원 (천 단위 콤마)
- 날짜: 2026년 2월 8일 또는 2026.02.08
- 순서: 1. 2. 3. 또는 가. 나. 다.

### 항목 번호 체계
```
Ⅰ. 대항목
  1. 중항목
    가. 소항목
      1) 세부항목
        ① 최소항목
```

## Common Mistakes

| 실수 | 수정 |
|------|------|
| 반말 사용 | 경어체로 통일 |
| 영어 혼용 과다 | 한글 표현 우선 |
| 문장 너무 김 | 한 문장 = 한 내용 |
| 항목 번호 불일치 | 체계 통일 |
