# 스킬 사용 가이드

<CRITICAL>
스킬이 적용될 가능성이 1%라도 있다면, 반드시 해당 스킬을 사용하세요.
이것은 선택이 아닙니다. 협상 불가. 예외 없음.
</CRITICAL>

## 핵심 규칙

**응답하기 전에 관련 스킬을 먼저 확인하세요.**

```
사용자 메시지 수신
    ↓
적용 가능한 스킬이 있는가? (1%라도)
    ↓ YES
스킬 로드 및 따르기
    ↓
응답
```

## Red Flags

이런 생각이 들면 **STOP** - 스킬 회피 중:

| 생각 | 현실 |
|------|------|
| "이건 간단한 질문이라..." | 질문도 작업이다. 스킬 체크. |
| "먼저 정보 좀 모아야..." | 스킬이 정보 수집 방법을 알려준다. |
| "스킬은 좀 과한데..." | 간단한 것도 복잡해진다. 스킬 사용. |
| "이 스킬 알고 있는데..." | 스킬은 진화한다. 최신 버전 읽기. |
| "이건 작업이 아닌데..." | 행동 = 작업. 스킬 체크. |
| "빨리 답하는 게 좋겠다..." | 정확한 답 > 빠른 답. 스킬 따르기. |

## 스킬 매칭 가이드

### 파일이 있을 때
| 파일 유형 | 스킬 |
|----------|------|
| .hwp, .hwpx | hwp-parsing |
| .xlsx, .xls, .csv | excel-analysis |
| .pdf | pdf-reading |
| .pptx, .ppt | ppt-analysis |

### 작성 요청
| 요청 | 스킬 |
|------|------|
| 공문, 보고서, 기안서 | document-drafting |
| 이메일, 메일 | email-writing |
| 회의록, 미팅 노트 | meeting-notes |
| 제안서, 사업계획서 | proposal-writing |

### 분석 요청
| 요청 | 스킬 |
|------|------|
| 데이터 분석 | excel-analysis |
| 요약해줘 | text-summary |
| 비교해줘 | comparison |

## 스킬 우선순위

여러 스킬이 적용 가능할 때:

1. **파일 처리 스킬** 먼저 (입력 처리)
2. **분석 스킬** 다음 (데이터 가공)
3. **작성 스킬** 마지막 (출력 생성)

## 스킬 유형

### 엄격한 스킬 (Rigid)
정확히 따라야 함. 적용 안 함 = 규칙 위반.
- document-drafting (양식 준수)
- meeting-notes (필수 항목)

### 유연한 스킬 (Flexible)
원칙을 맥락에 맞게 적용.
- text-summary (길이 조절)
- email-writing (톤 조절)
